Python ile API Geliştirme

Bu dersimizde Python'da API nedir, nasıl geliştirilir ve farklı API türleri hakkında bilgi edineceksiniz.

API Nedir?

API (Application Programming Interface), uygulamaların birbirleriyle iletişim kurmasını sağlayan bir arayüzdür. Bir API, uygulamanın özelliklerini veya verilerini başka uygulamalarla paylaşmak için tanımlanmış bir dizi kural ve protokoldür.

Python, API geliştirmek için mükemmel bir dildir çünkü zengin kütüphane ekosistemi ve okunabilir sözdizimi ile hızlı geliştirme yapmanıza olanak tanır.

# Basit bir Flask API örneği from flask import Flask, jsonify app = Flask(__name__) @app.route('/api/hello', methods=['GET']) def hello(): return jsonify({"message": "Merhaba, API Dünyası!"}) if __name__ == '__main__': app.run(debug=True)

Biliyor muydunuz?

Web API'leri genellikle REST (Representational State Transfer) mimarisi kullanılarak tasarlanır, ancak gRPC, GraphQL gibi farklı API modelleri de yaygın olarak kullanılmaktadır.

API Tasarım Prensipleri

İyi bir API tasarımı için göz önünde bulundurulması gereken bazı prensipler vardır:

  • Tutarlılık: API'nizin tüm bölümleri benzer şekilde davranmalıdır.
  • Basitlik: API'niz mümkün olduğunca basit ve anlaşılır olmalıdır.
  • Dokümantasyon: API'niz iyi belgelenmiş olmalıdır.
  • Güvenlik: API'niz güvenli olmalı ve yetkilendirme içermelidir.
  • Versiyonlama: Değişiklikler için API versiyonlama stratejisi olmalıdır.
# RESTful endpoint tasarım örneği @app.route('/api/v1/users', methods=['GET']) def get_users(): # Tüm kullanıcıları getir return jsonify(users) @app.route('/api/v1/users/', methods=['GET']) def get_user(user_id): # Belirli bir kullanıcıyı getir return jsonify(users.get(user_id)) @app.route('/api/v1/users', methods=['POST']) def create_user(): # Yeni kullanıcı oluştur return jsonify({"status": "created"}), 201 @app.route('/api/v1/users/', methods=['PUT']) def update_user(user_id): # Kullanıcıyı güncelle return jsonify({"status": "updated"}) @app.route('/api/v1/users/', methods=['DELETE']) def delete_user(user_id): # Kullanıcıyı sil return jsonify({"status": "deleted"}), 204

Python'da API Geliştirme Kütüphaneleri

Python'da API geliştirmek için kullanabileceğiniz birçok kütüphane vardır:

1. Flask

  • Mikro çerçeve: Hafif ve esnek, küçük ve orta ölçekli API'ler için idealdir
  • Hızlı başlangıç: Minimum yapılandırma ile hızlıca API geliştirmeye başlayabilirsiniz
  • Genişletilebilir: İhtiyacınıza göre eklentilerle genişletilebilir
# Flask ile basit bir API from flask import Flask, request, jsonify app = Flask(__name__) # Basit bir veritabanı yerine kullanılacak liste tasks = [ {"id": 1, "title": "Alışveriş yap", "done": False}, {"id": 2, "title": "Raporu tamamla", "done": True} ] @app.route('/api/tasks', methods=['GET']) def get_tasks(): return jsonify({"tasks": tasks}) @app.route('/api/tasks/', methods=['GET']) def get_task(task_id): task = next((task for task in tasks if task["id"] == task_id), None) if task: return jsonify({"task": task}) return jsonify({"error": "Task not found"}), 404 @app.route('/api/tasks', methods=['POST']) def create_task(): if not request.json or 'title' not in request.json: return jsonify({"error": "Title field is required"}), 400 task = { "id": tasks[-1]["id"] + 1 if tasks else 1, "title": request.json['title'], "done": request.json.get('done', False) } tasks.append(task) return jsonify({"task": task}), 201 if __name__ == '__main__': app.run(debug=True)

2. Django REST Framework

  • Güçlü ve kapsamlı: Django'nun üzerine inşa edilmiş, tam özellikli bir REST API çerçevesi
  • Model tabanlı: Django modelleri üzerinden otomatik API oluşturma
  • Gelişmiş özellikler: Kimlik doğrulama, izinler, throttling, viewset'ler ve daha fazlası
# Django REST Framework örneği # models.py from django.db import models class Task(models.Model): title = models.CharField(max_length=100) description = models.TextField(blank=True) done = models.BooleanField(default=False) created_at = models.DateTimeField(auto_now_add=True) def __str__(self): return self.title # serializers.py from rest_framework import serializers from .models import Task class TaskSerializer(serializers.ModelSerializer): class Meta: model = Task fields = ['id', 'title', 'description', 'done', 'created_at'] # views.py from rest_framework import viewsets from .models import Task from .serializers import TaskSerializer class TaskViewSet(viewsets.ModelViewSet): queryset = Task.objects.all() serializer_class = TaskSerializer # urls.py from django.urls import path, include from rest_framework.routers import DefaultRouter from .views import TaskViewSet router = DefaultRouter() router.register(r'tasks', TaskViewSet) urlpatterns = [ path('api/', include(router.urls)), ]

3. FastAPI

  • Modern ve hızlı: ASGI tabanlı, yüksek performanslı framework
  • Otomatik dokümantasyon: Swagger ve ReDoc ile API dokümantasyonu
  • Tip kontrolü: Python type hints ile güçlü veri doğrulama
# FastAPI ile API geliştirme from fastapi import FastAPI, HTTPException from pydantic import BaseModel from typing import List, Optional app = FastAPI(title="Task API") class TaskBase(BaseModel): title: str description: Optional[str] = None done: bool = False class TaskCreate(TaskBase): pass class Task(TaskBase): id: int class Config: orm_mode = True # Geçici veritabanı db = [] counter = 0 @app.post("/api/tasks/", response_model=Task, status_code=201) def create_task(task: TaskCreate): global counter counter += 1 new_task = Task(id=counter, **task.dict()) db.append(new_task) return new_task @app.get("/api/tasks/", response_model=List[Task]) def read_tasks(): return db @app.get("/api/tasks/{task_id}", response_model=Task) def read_task(task_id: int): task = next((task for task in db if task.id == task_id), None) if task is None: raise HTTPException(status_code=404, detail="Task not found") return task

4. Connexion

  • OpenAPI spesifikasyonu: Swagger/OpenAPI spesifikasyonuna göre API tasarımı
  • Otomatik yönlendirme: API dokümantasyonundan kod yerine koddan dokümantasyon
  • Şema doğrulama: İstek ve yanıt doğrulama
# Connexion ile API geliştirme import connexion from connexion import NoContent # Veritabanı yerine geçici liste TASKS = {} def get_tasks(): return [task for task in TASKS.values()], 200 def get_task(task_id): task = TASKS.get(task_id) return task or NoContent, 200 if task else 404 def create_task(task): task_id = len(TASKS) + 1 task['id'] = task_id TASKS[task_id] = task return task, 201 def update_task(task_id, task): if task_id in TASKS: task['id'] = task_id TASKS[task_id] = task return TASKS[task_id], 200 return NoContent, 404 def delete_task(task_id): if task_id in TASKS: del TASKS[task_id] return NoContent, 204 return NoContent, 404 # API uygulamasını başlat app = connexion.App(__name__, specification_dir='./swagger/') app.add_api('swagger.yaml') if __name__ == '__main__': app.run(port=8080)

API Güvenliği

API'nizi güvenli hale getirmek için aşağıdaki yöntemleri kullanabilirsiniz:

# Flask ile JWT tabanlı kimlik doğrulama from flask import Flask, jsonify, request from flask_jwt_extended import JWTManager, create_access_token, jwt_required, get_jwt_identity app = Flask(__name__) app.config['JWT_SECRET_KEY'] = 'gizli-anahtar-buraya' # Gerçek uygulamada daha güvenli bir yöntemle saklanmalıdır jwt = JWTManager(app) # Basit kullanıcı veritabanı users = { 'admin': { 'password': 'password123', # Gerçek uygulamada hash'lenmiş olmalıdır 'role': 'admin' } } @app.route('/api/login', methods=['POST']) def login(): if not request.is_json: return jsonify({"error": "JSON verisi bekleniyor"}), 400 username = request.json.get('username', None) password = request.json.get('password', None) if not username or not password: return jsonify({"error": "Kullanıcı adı ve şifre gerekli"}), 400 user = users.get(username, None) if not user or user['password'] != password: return jsonify({"error": "Geçersiz kullanıcı adı veya şifre"}), 401 # Token oluştur access_token = create_access_token(identity=username) return jsonify(access_token=access_token), 200 @app.route('/api/protected', methods=['GET']) @jwt_required() def protected(): # Mevcut kullanıcının kimliğini al current_user = get_jwt_identity() return jsonify(logged_in_as=current_user), 200 if __name__ == '__main__': app.run(debug=True)

API Güvenlik İpuçları

API güvenliğini artırmak için şu önlemleri alabilirsiniz: HTTPS kullanın, rate limiting uygulayın, API anahtarları veya JWT token'ları kullanın, input doğrulama yapın ve sürekli güvenlik güncellemeleri yapın.

# Flask-Limiter ile rate limiting örneği from flask import Flask from flask_limiter import Limiter from flask_limiter.util import get_remote_address app = Flask(__name__) limiter = Limiter( app, key_func=get_remote_address, default_limits=["200 per day", "50 per hour"] ) @app.route("/api/limited") @limiter.limit("5 per minute") def limited_resource(): return "Bu endpoint dakikada en fazla 5 istek alabilir."
