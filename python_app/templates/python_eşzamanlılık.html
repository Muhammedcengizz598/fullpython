Python'da Eşzamanlılık

Bu dersimizde Python'da eşzamanlılık (concurrency) nedir, nasıl uygulanır ve modern Python programlamada Thread, Multiprocessing ve Asenkron programlama hakkında bilgi edineceksiniz.

Eşzamanlılık Nedir?

Eşzamanlılık, birden fazla işin aynı anda yürütülmesi veya ilerlemesi anlamına gelir. Python'da eşzamanlılık, programınızın verimini artırmak ve özellikle I/O bağlantılı işlemler sırasında bekleme sürelerini azaltmak için kullanılır.

Python'da eşzamanlılık için üç temel yaklaşım bulunmaktadır:

  • Thread (İş Parçacığı): Hafif, ortak bellek paylaşımlı eşzamanlılık
  • Multiprocessing (Çok İşlemli): Ayrı işlemler kullanarak paralel çalışma
  • Asynchronous Programming (Asenkron Programlama): Tek iş parçacığında eşzamanlı çalışma

Biliyor muydunuz?

Python'da Global Interpreter Lock (GIL) nedeniyle, thread kullansanız bile CPU yoğun işlemler gerçekten paralel çalışmaz. Gerçek paralellik için multiprocessing kullanmanız gerekir.

1. Threading (İş Parçacığı)

Thread'ler, aynı işlem içinde çalışan hafif yürütme birimleridir. Python'da threading modülü ile kolayca thread'ler oluşturabilirsiniz.

import threading import time def fonksiyon(id): print(f"Thread {id} başladı") time.sleep(2) # I/O işlemini simüle ediyoruz print(f"Thread {id} bitti") # Thread'leri başlatma threads = [] for i in range(5): t = threading.Thread(target=fonksiyon, args=(i,)) threads.append(t) t.start() # Ana thread, tüm thread'lerin bitmesini bekliyor for t in threads: t.join() print("Tüm thread'ler tamamlandı")

Thread'lerin avantajları ve dezavantajları:

  • Avantajlar: Hafiftir, ortak bellek erişimi sağlar, I/O işlemlerinde verimlidir.
  • Dezavantajlar: GIL nedeniyle CPU-bound işlemlerde gerçek paralellik sağlamaz, thread güvenliği sorunları olabilir.

Thread Senkronizasyonu

Thread'ler aynı verilere eriştiğinde yarış koşulları (race conditions) oluşabilir. Bunu önlemek için lock mekanizmaları kullanılır:

import threading import time sayac = 0 lock = threading.Lock() def arttir(): global sayac for _ in range(100000): with lock: # lock.acquire() ve lock.release() işlemlerini otomatik yapar sayac += 1 # İki thread oluşturalım t1 = threading.Thread(target=arttir) t2 = threading.Thread(target=arttir) t1.start() t2.start() t1.join() t2.join() print(f"Sayaç değeri: {sayac}") # 200000 olmasını bekliyoruz

2. Multiprocessing (Çok İşlemli)

Python'un multiprocessing modülü, birden fazla işlem başlatarak GIL'i aşmanızı ve çok çekirdekli işlemcilerden tam olarak yararlanmanızı sağlar.

import multiprocessing import time def yoğun_hesaplama(sayı): print(f"İşlem {sayı} başladı") # CPU-bound işlemi simüle ediyoruz sonuç = 0 for i in range(10000000): sonuç += i print(f"İşlem {sayı} bitti, sonuç: {sonuç}") return sayı, sonuç if __name__ == "__main__": # Windows'ta multiprocessing için gerekli # İşlem havuzu oluştur with multiprocessing.Pool(processes=4) as havuz: # İşlemleri eşzamanlı başlat sonuçlar = havuz.map(yoğun_hesaplama, range(8)) print("Tüm işlemler tamamlandı") print(f"Sonuçlar: {sonuçlar}")

Multiprocessing'in avantajları ve dezavantajları:

  • Avantajlar: GIL'i aşarak gerçek paralellik sağlar, CPU-bound işlemlerde idealdir.
  • Dezavantajlar: Thread'lerden daha fazla kaynak kullanır, işlemler arası veri paylaşımı daha karmaşıktır.

İşlemler Arası İletişim

İşlemler arasında veri paylaşmak için Queue, Pipe veya Manager gibi yapılar kullanabilirsiniz:

import multiprocessing import time def üretici(q): for i in range(5): q.put(i) print(f"Üreten: {i}") time.sleep(0.5) def tüketici(q): while True: değer = q.get() if değer is None: # Bitiş sinyali break print(f"Tüketen: {değer}") if __name__ == "__main__": # Sıra oluştur kuyruk = multiprocessing.Queue() # İşlemleri başlat p1 = multiprocessing.Process(target=üretici, args=(kuyruk,)) p2 = multiprocessing.Process(target=tüketici, args=(kuyruk,)) p1.start() p2.start() p1.join() kuyruk.put(None) # Tüketici için bitiş sinyali p2.join()

3. Asenkron Programlama

Python 3.4'ten beri asyncio modülü ile asenkron programlama desteklenmektedir. Python 3.5 ve sonrasında daha da geliştirilmiş async/await sözdizimi ile kullanımı kolaylaşmıştır.

Asenkron programlama, tek bir thread üzerinde eşzamanlı işlemler yapmanıza olanak tanır. Bu özellikle I/O-bound işlemler için çok verimlidir.

import asyncio import time async def asenkron_görev(id): print(f"Görev {id} başladı") await asyncio.sleep(2) # I/O işlemini simüle ediyoruz print(f"Görev {id} bitti") return id async def main(): # Görevleri eşzamanlı çalıştır görevler = [asenkron_görev(i) for i in range(5)] sonuçlar = await asyncio.gather(*görevler) print(f"Tüm görevler tamamlandı: {sonuçlar}") # Event loop oluştur ve çalıştır if __name__ == "__main__": başlangıç = time.time() asyncio.run(main()) bitiş = time.time() print(f"Toplam süre: {bitiş - başlangıç:.2f} saniye") # ~2 saniye

Senkron olarak çalıştırsaydık, toplam süre 10 saniye olacaktı (5 görev x 2 saniye), ancak asenkron çalıştırdığımızda yaklaşık 2 saniye sürer.

Asenkron Context Manager

Python 3.7'den itibaren, async with deyimi kullanarak asenkron bağlamları yönetebilirsiniz:

import asyncio class AsenkronKaynakYöneticisi: async def __aenter__(self): print("Kaynak açılıyor...") await asyncio.sleep(1) # Kaynak açılmasını simüle ediyoruz return self async def __aexit__(self, exc_type, exc_val, exc_tb): print("Kaynak kapatılıyor...") await asyncio.sleep(0.5) # Kaynak kapatılmasını simüle ediyoruz async def işlem_yap(self): print("Kaynak üzerinde işlem yapılıyor...") await asyncio.sleep(1) async def main(): async with AsenkronKaynakYöneticisi() as kaynak: await kaynak.işlem_yap() if __name__ == "__main__": asyncio.run(main())

Asenkron Döngüler

Python 3.8'den itibaren, async for deyimi kullanarak asenkron iteratorler üzerinde döngü yapabilirsiniz:

import asyncio class AsenkronSayıÜreteci: def __init__(self, son): self.son = son self.sayı = 0 def __aiter__(self): return self async def __anext__(self): if self.sayı >= self.son: raise StopAsyncIteration değer = self.sayı self.sayı += 1 await asyncio.sleep(0.5) # İşlemi simüle ediyoruz return değer async def main(): async for sayı in AsenkronSayıÜreteci(5): print(f"Sayı: {sayı}") if __name__ == "__main__": asyncio.run(main())

Ne Zaman Hangi Yaklaşımı Kullanmalı?

Doğru eşzamanlılık yaklaşımını seçmek, çözdüğünüz probleme bağlıdır:

  • Threading: I/O-bound işlemler için uygundur (dosya I/O, ağ istekleri) ve paylaşılan bellek erişimi gerektiğinde.
  • Multiprocessing: CPU-bound işlemler için uygundur ve gerçek paralellik gerektiğinde.
  • Asyncio: Çok sayıda I/O-bound işlemi çok sayıda thread oluşturmadan yönetmek istediğinizde idealdir.

Pratik İpucu

Thread ve multiprocessing kullanırken, işin sonuçlarını toplamak için concurrent.futures modülü kullanmak işinizi kolaylaştırabilir:

from concurrent.futures import ThreadPoolExecutor import time def uzun_süren_iş(id): time.sleep(2) return f"İş {id} tamamlandı" # ThreadPoolExecutor kullanımı (ProcessPoolExecutor da benzer şekilde kullanılır) with ThreadPoolExecutor(max_workers=5) as executor: sonuçlar = list(executor.map(uzun_süren_iş, range(10))) print(sonuçlar)

thread2 = threading.Thread(target=print_numbers)

thread1.start()
thread2.start()

thread1.join()
thread2.join()

B

Doğru! Thread'ler eşzamanlı çalışır ve işletim sistemi tarafından sıralanır. Bu nedenle, her çalıştırmada thread'ler farklı sıralarda çalışabilir ve çıktıda sayıların karışık bir sırada görülmesi normaldir.
Yanlış. Thread'ler eşzamanlı çalışır ve işletim sistemi tarafından sıralanır. Bu nedenle, her çalıştırmada thread'ler farklı sıralarda çalışabilir ve çıktıda sayıların karışık bir sırada görülmesi normaldir.

Sonraki Adımlar

Python eşzamanlılık ve paralel programlama konusunu tamamladıktan sonra öğrenmenize devam edebileceğiniz konular:

  • Gelişmiş asyncio: Asenkron içerik yöneticileri, asenkron üreteciler ve asenkron döngüler.
  • Paralel Veri İşleme: Büyük veri setlerini işleme ve dask gibi kütüphaneleri kullanma.
  • Eşzamanlı Web İstekleri: aiohttp ve requests-futures kullanarak hızlı web istekleri yapma.

  • Veri Paylaşımı ve Senkronizasyon: Lock, Semaphore, Queue ve diğer senkronizasyon mekanizmaları.

Başarı İpucu

Python'da eşzamanlılık konusunu öğrenirken, önce problemin I/O-bound mu yoksa CPU-bound mu olduğunu anlamak çok önemlidir. I/O-bound işlemler için asyncio veya threading, CPU-bound işlemler için ise multiprocessing kullanmak en iyi performansı almanızı sağlayacaktır!
