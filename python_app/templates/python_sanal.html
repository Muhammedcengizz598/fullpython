Python'da Sanal Ortamlar

Bu dersimizde Python'da sanal ortamlar nedir, neden kullanılır ve nasıl oluşturulur konularını öğreneceksiniz.

Sanal Ortamlar Nedir?

Python sanal ortamları, izole Python çalışma ortamları oluşturmanızı sağlayan araçlardır. Bir sanal ortam, farklı projeleriniz için farklı bağımlılık ve paket sürümlerini ayrı tutmanızı sağlar.

Sanal ortamlar, projenizin ihtiyaç duyduğu kütüphane sürümlerini diğer projelerden izole etmenize yardımcı olur. Bu şekilde, bir projede yaptığınız değişiklikler diğer projeleri etkilemez.

# Sanal ortam olmadan paket yükleme (global) pip install requests # Global yüklenen paketler tüm projeleriniz için aynı sürümü kullanır # Bu bazen uyumluluk sorunlarına yol açabilir

Biliyor muydunuz?

Sanal ortamlar sayesinde aynı bilgisayarda farklı Python sürümleri ve paket sürümleriyle çalışabilirsiniz.

Neden Sanal Ortamlar Kullanmalıyız?

Sanal ortamlar kullanmanın birçok avantajı vardır:

  • Bağımlılık çakışmalarını önler: Farklı projeler farklı kütüphane sürümlerine ihtiyaç duyabilir.
  • Temiz bir geliştirme ortamı sağlar: Sadece ihtiyacınız olan paketleri içerir.
  • Projenizi taşınabilir hale getirir: Sanal ortamınızı requirements.txt dosyası ile kolayca paylaşabilirsiniz.
  • Sistem genelindeki Python kurulumunuzu korur: Global Python ortamınızı değiştirmek zorunda kalmazsınız.
  • Test ve geliştirme ortamlarını ayırmanızı sağlar: Farklı ortamlarda farklı konfigürasyonlar kullanabilirsiniz.
# Örnek senaryo: # Proje A: Django 2.2 gerektirir # Proje B: Django 3.2 gerektirir # Sanal ortam olmadan bu iki projeyi aynı # bilgisayarda çalıştırmak sorun yaratabilir

Python'da Sanal Ortam Oluşturma

Python'da sanal ortam oluşturmak için farklı araçlar vardır:

1. venv (Python 3.3+)

venv, Python'un resmi ve standart sanal ortam oluşturma aracıdır. Python 3.3 ve sonraki sürümlerde kullanılabilir.

# Sanal ortam oluşturma python -m venv myenv # Windows'ta sanal ortamı aktifleştirme myenv\Scripts\activate # Linux/macOS'ta sanal ortamı aktifleştirme source myenv/bin/activate # Sanal ortam aktif olduğunda, komut satırı genellikle # şu şekilde görünür: (myenv) C:\Users\kullanici> # Sanal ortamı devre dışı bırakmak için deactivate

2. virtualenv

virtualenv, venv'den daha eski ancak daha fazla özellik sunan bir sanal ortam aracıdır. Python 2 ve 3 ile çalışır.

# virtualenv yükleme pip install virtualenv # Sanal ortam oluşturma virtualenv myenv # Aktivasyon ve deaktivasyon venv ile aynıdır # Windows: myenv\Scripts\activate # Linux/macOS: source myenv/bin/activate # Deaktive etmek için: deactivate

3. conda

conda, Anaconda dağıtımıyla gelen ve sadece Python paketlerini değil, diğer dillere ait paketleri de yönetebilen bir paket ve ortam yönetim sistemidir.

# Conda ile sanal ortam oluşturma conda create --name myenv python=3.9 # Conda ortamını aktifleştirme # Windows: conda activate myenv # Linux/macOS: conda activate myenv # Deaktive etmek için: conda deactivate

Sanal Ortamlarda Paket Yönetimi

Sanal ortamlarla çalışırken, paketleri yönetmek için genellikle pip kullanılır:

# Sanal ortam aktifken paket yükleme pip install requests # Belirli bir sürüm yükleme pip install requests==2.26.0 # Yüklü paketleri listeleme pip list # Paket kaldırma pip uninstall requests # Yüklü paketleri requirements.txt'ye kaydetme pip freeze > requirements.txt # requirements.txt'den paketleri yükleme pip install -r requirements.txt

Proje Bağımlılıklarını Yönetme

Bir projenin bağımlılıklarını requirements.txt dosyasında tutmak, projenizi başka bilgisayarlarda veya sunucularda kolayca kurmanızı sağlar. Bu dosya genellikle projenizin kök dizininde bulunur.

# requirements.txt örneği requests==2.26.0 Django==3.2.9 numpy==1.21.4 matplotlib==3.5.0

Farklı Python Sürümleriyle Çalışma

Sanal ortamlar, farklı Python sürümleriyle çalışmanıza olanak tanır:

# virtualenv ile belirli bir Python sürümü kullanmak virtualenv -p python3.8 myenv # conda ile belirli bir Python sürümü kullanmak conda create --name myenv python=3.8

Sanal Ortamları Proje Klasörü İçinde Tutma

Sanal ortamları proje klasörü içinde tutmak yaygın bir uygulamadır:

# Proje dizininde sanal ortam oluşturma mkdir my_project cd my_project python -m venv venv # Bu durumda dosya yapısı şöyle olur: # my_project/ # ├── venv/ (sanal ortam dosyaları) # ├── src/ (kaynak kodunuz) # ├── requirements.txt # └── README.md

Dikkat

Sanal ortam klasörlerini (.venv, venv, env gibi) git gibi sürüm kontrol sistemlerine eklememek en iyi uygulamadır. Bunun yerine, .gitignore dosyanıza bu klasörleri ekleyin.

Popüler Sanal Ortam Yönetim Araçları

Daha gelişmiş sanal ortam yönetimi için ek araçlar da mevcuttur:

1. pipenv

pipenv, pip ve virtualenv'i birleştiren ve bağımlılık yönetimini geliştirilmiş bir şekilde sunan bir araçtır.

# pipenv yükleme pip install pipenv # Pipenv ile proje başlatma pipenv install # Belirli bir paket yükleme pipenv install requests # Sadece geliştirme ortamı için paket yükleme pipenv install pytest --dev # Pipenv ortamını aktifleştirme pipenv shell # Pipenv ortamında komut çalıştırma (aktive etmeden) pipenv run python script.py

2. poetry

poetry, Python projelerinin bağımlılıklarını yönetmek ve paketlemek için modern bir araçtır.

# Poetry yükleme pip install poetry # Yeni bir proje oluşturma poetry new my-project # Mevcut bir projede poetry başlatma cd my-project poetry init # Bağımlılık ekleme poetry add requests # Geliştirme bağımlılığı ekleme poetry add pytest --dev # Poetry ortamını aktifleştirme poetry shell # Ortamda komut çalıştırma poetry run python script.py

3. pyenv

pyenv, birden fazla Python sürümünü aynı sistemde yönetmenize olanak tanır.

# macOS/Linux için pyenv yükleme (Homebrew ile) brew install pyenv # Kullanılabilir Python sürümlerini listeleme pyenv install --list # Belirli bir Python sürümünü yükleme pyenv install 3.9.7 # Sisteme özgü Python sürümünü değiştirme pyenv global 3.9.7 # Proje için Python sürümünü değiştirme cd my-project pyenv local 3.8.10

Kendinizi Test Edin

Python sanal ortamlar konusunu ne kadar anladığınızı test etmek için aşağıdaki soruları cevaplayın:

1. Python sanal ortamlarını kullanmanın ana amacı nedir?

A
Proje bağımlılıklarını izole etmek
B
Python kodunu daha hızlı çalıştırmak
C
Disk alanından tasarruf etmek
D
Program güvenliğini artırmak
Doğru! Sanal ortamlar, proje bağımlılıklarını izole ederek farklı projelerin farklı paket sürümleri kullanabilmesini sağlar.
Yanlış. Sanal ortamların ana amacı, projelerin bağımlılıklarını birbirinden izole etmektir, böylece her proje kendi paket sürümlerini kullanabilir.

2. Aşağıdaki komutlardan hangisi bir sanal ortam oluşturmak için kullanılır?

A
pip install venv
B
python -m venv myenv
C
python venv --create
D
pip venv start
Doğru! "python -m venv myenv" komutu, Python'un yerleşik venv modülünü kullanarak "myenv" adında bir sanal ortam oluşturur.
Yanlış. Sanal ortam oluşturmak için doğru komut "python -m venv myenv" şeklindedir.

3. Windows'ta bir sanal ortamı aktifleştirmek için hangi komut kullanılır?

A
myenv\Scripts\activate
B
source myenv/bin/activate
C
python myenv/activate.py
D
start myenv
Doğru! Windows'ta sanal ortamı aktifleştirmek için "myenv\Scripts\activate" komutu kullanılır.
Yanlış. Windows'ta sanal ortamı aktifleştirmek için "myenv\Scripts\activate" komutu kullanılır. "source myenv/bin/activate" komutu Linux ve macOS için kullanılır.

4. Bir sanal ortamdaki yüklü paketleri listelemek için hangi komut kullanılır?

A
pip show
B
pip list
C
pip installed
D
pip display
Doğru! "pip list" komutu, aktif sanal ortamda yüklü olan tüm paketleri listeler.
Yanlış. Yüklü paketleri listelemek için "pip list" komutu kullanılır. "pip show" belirli bir paketin detaylarını gösterir.

5. Sanal ortamdaki paketleri requirements.txt dosyasına kaydetmek için hangi komut kullanılır?

A
pip save > requirements.txt
B
pip freeze > requirements.txt
C
pip list > requirements.txt
D
pip requirements --save
Doğru! "pip freeze > requirements.txt" komutu, yüklü paketleri ve sürümlerini bir requirements.txt dosyasına kaydeder.
Yanlış. Yüklü paketleri requirements.txt dosyasına kaydetmek için "pip freeze > requirements.txt" komutu kullanılır.

6. Aşağıdakilerden hangisi Python sanal ortam aracı DEĞİLDİR?

A
venv
B
virtualenv
C
pipconfig
D
conda
Doğru! "pipconfig" diye bir sanal ortam aracı yoktur. Diğer seçenekler (venv, virtualenv ve conda) gerçek Python sanal ortam araçlarıdır.
Yanlış. venv, virtualenv ve conda gerçek Python sanal ortam araçlarıdır, ancak "pipconfig" gerçek bir araç değildir.

7. Bir sanal ortamı devre dışı bırakmak için hangi komut kullanılır?

A
deactivate
B
exit
C
venv stop
D
quit venv
Doğru! Sanal ortamı devre dışı bırakmak için "deactivate" komutu kullanılır.
Yanlış. Sanal ortamı devre dışı bırakmak için "deactivate" komutu kullanılır.

8. Aşağıdaki komut Python sanal ortamında ne yapar?

python -m venv myenv
A
Var olan bir sanal ortamı aktifleştirir
B
Sanal ortamdan çıkış yapar
C
Yeni bir sanal ortam oluşturur
D
Sanal ortamdaki paketleri listeler
Doğru! Bu komut "myenv" adında yeni bir Python sanal ortamı oluşturur. Sanal ortam, projeniz için izole bir Python ortamı sağlar.
Yanlış. Bu komut "myenv" adında yeni bir Python sanal ortamı oluşturur. Sanal ortamı aktifleştirmek için Windows'ta "myenv\Scripts\activate", Unix/MacOS'ta "source myenv/bin/activate" komutunu kullanmanız gerekir.

9. Python sanal ortamlar için aşağıdakilerden hangisi doğrudur?

A
Sanal ortamlar farklı Python sürümleri kullanamaz
B
Sanal ortamlar sadece Windows işletim sisteminde çalışır
C
Sanal ortamlar projeleri paket çakışmalarından korur
D
Sanal ortamlar internet bağlantısı olmadan kullanılamaz
Doğru! Python sanal ortamları, farklı projelerin farklı paket gereksinimleri olduğunda paketlerin ve bağımlılıkların çakışmasını önler. Her projenin kendi izole ortamını oluşturarak bağımlılık çakışmalarını engeller.
Yanlış. Python sanal ortamları, farklı projelerin farklı paket gereksinimleri olduğunda paketlerin ve bağımlılıkların çakışmasını önlemek için kullanılır. Ayrıca tüm işletim sistemlerinde çalışır ve internet bağlantısı olmadan da kullanılabilir.

10. Aşağıdakilerden hangisi Python sanal ortamını devre dışı bırakma komutudur?

A
python -m venv deactivate
B
exit venv
C
python -m deactivate
D
deactivate
Doğru! Aktif bir sanal ortamdan çıkmak için "deactivate" komutunu kullanmanız yeterlidir. Bu komut, terminal oturumunuzu normal Python ortamına geri döndürür.
Yanlış. Aktif bir sanal ortamdan çıkmak için sadece "deactivate" komutunu kullanmanız yeterlidir. Diğer seçenekler geçerli komutlar değildir.

Python sanal ortamlar konusunu tamamladıktan sonra öğrenmenize devam edebileceğiniz konular:
