Python'da Test Yazımı

Bu dersimizde Python'da test yazma, birim testleri, entegrasyon testleri ve test güdümlü geliştirme hakkında bilgi edineceksiniz.

Test Nedir ve Neden Önemlidir?

Yazılım testleri, kodunuzun beklendiği gibi çalıştığını doğrulamak için kullanılan yöntemlerdir. İyi yazılmış testler, kodunuzdaki hataları erkenden yakalamak, kodun kalitesini artırmak ve değişiklikler yapıldığında mevcut işlevselliğin bozulmadığından emin olmak için hayati önemdedir.

Python'da test yazımı için zengin bir ekosistem bulunmaktadır. Standart kütüphanedeki unittest modülünden başlayarak pytest ve nose gibi popüler üçüncü taraf kütüphanelere kadar çeşitli araçlar mevcuttur.

# Basit bir fonksiyon tanımlama def topla(a, b): return a + b # unittest kullanarak test yazma import unittest class TestToplama(unittest.TestCase): def test_pozitif_sayilar(self): self.assertEqual(topla(3, 5), 8) def test_negatif_sayilar(self): self.assertEqual(topla(-1, -1), -2) def test_karisik_sayilar(self): self.assertEqual(topla(-1, 1), 0) if __name__ == '__main__': unittest.main()

Biliyor muydunuz?

İyi test yazımı, yazılım projelerinde hataları %40 ila %80 oranında azaltabilir ve uzun vadede bakım maliyetlerini düşürebilir.

Python'da Test Araçları

Python'da en yaygın kullanılan test araçları şunlardır:

  • unittest: Python'un standart kütüphanesinde bulunan test çerçevesi
  • pytest: Daha basit sözdizimi ve güçlü özellikler sunan popüler bir test çerçevesi
  • nose/nose2: unittest'i genişleten test keşif ve yürütme araçları
  • doctest: Dökümantasyon içine yerleştirilen testler için araç
  • mock: Test yaparken nesnelerin davranışlarını taklit etmek için kullanılan kütüphane
# pytest kullanarak test yazma - daha öz bir sözdizimi def topla(a, b): return a + b # test_ ile başlayan fonksiyonlar pytest tarafından otomatik algılanır def test_topla_fonksiyonu(): assert topla(3, 5) == 8 assert topla(-1, -1) == -2 assert topla(-1, 1) == 0

Test Türleri

Python'da yaygın olarak kullanılan test türleri şunlardır:

1. Birim Testleri (Unit Tests)

  • Amaç: Kodun küçük, izole parçalarının (genellikle fonksiyonlar veya metotlar) beklendiği gibi çalıştığını doğrulamak
  • Kapsam: Tek bir fonksiyon veya metot
  • Bağımlılıklar: Genellikle mock (sahte) nesneler kullanılarak dış bağımlılıklar simüle edilir
# Bir sınıf için birim testi örneği class Hesaplama: def carp(self, a, b): return a * b def bol(self, a, b): if b == 0: raise ValueError("Sıfıra bölünemez") return a / b # unittest ile test import unittest class TestHesaplama(unittest.TestCase): def setUp(self): # Her test metodundan önce çalışır self.hesap = Hesaplama() def test_carp(self): self.assertEqual(self.hesap.carp(3, 4), 12) self.assertEqual(self.hesap.carp(-2, 5), -10) def test_bol(self): self.assertEqual(self.hesap.bol(10, 2), 5) # Hata fırlatma durumunu test etme with self.assertRaises(ValueError): self.hesap.bol(5, 0)

2. Entegrasyon Testleri (Integration Tests)

  • Amaç: Birden fazla bileşenin birlikte çalıştığını doğrulamak
  • Kapsam: Birbirine bağlı birkaç bileşenin işlevselliği
  • Bağımlılıklar: Gerçek bağımlılıkları kullanabilir veya dış sistemler için sahte sunucular (mock server) oluşturabilir
# Veritabanı bağlantısı gerektiren bir sınıf import sqlite3 class VeriTabaniIslemleri: def __init__(self, db_adi): self.baglanti = sqlite3.connect(db_adi) self.cursor = self.baglanti.cursor() # Tablo oluştur self.cursor.execute(''' CREATE TABLE IF NOT EXISTS kullanicilar (id INTEGER PRIMARY KEY, ad TEXT, yas INTEGER) ''') self.baglanti.commit() def kullanici_ekle(self, ad, yas): self.cursor.execute("INSERT INTO kullanicilar (ad, yas) VALUES (?, ?)", (ad, yas)) self.baglanti.commit() return self.cursor.lastrowid def kullanici_getir(self, kullanici_id): self.cursor.execute("SELECT * FROM kullanicilar WHERE id = ?", (kullanici_id,)) return self.cursor.fetchone() def kapat(self): self.baglanti.close() # Entegrasyon testi - gerçek bir SQLite veritabanı kullanılıyor import unittest import os class TestVeriTabaniIslemleri(unittest.TestCase): def setUp(self): # Test için geçici bir veritabanı oluştur self.test_db = "test.db" self.vt = VeriTabaniIslemleri(self.test_db) def tearDown(self): # Test sonrası temizlik self.vt.kapat() if os.path.exists(self.test_db): os.remove(self.test_db) def test_kullanici_ekle_ve_getir(self): # Kullanıcı ekle kullanici_id = self.vt.kullanici_ekle("Ahmet", 30) # Eklenen kullanıcıyı getir kullanici = self.vt.kullanici_getir(kullanici_id) # Doğrula self.assertIsNotNone(kullanici) self.assertEqual(kullanici[1], "Ahmet") self.assertEqual(kullanici[2], 30)

3. Fonksiyonel Testler (Functional Tests)

  • Amaç: Uygulamanın son kullanıcı açısından doğru çalıştığını doğrulamak
  • Kapsam: Tüm sistem veya belirli bir kullanıcı senaryosu
  • Bağımlılıklar: Genellikle gerçek sistemleri kullanır ancak test veritabanları gibi izole ortamlarla çalışabilir
# Selenium ile web uygulaması için fonksiyonel test örneği from selenium import webdriver import unittest import time class TestLoginSayfasi(unittest.TestCase): def setUp(self): # WebDriver'ı başlat self.driver = webdriver.Chrome() self.driver.get("http://example.com/login") def tearDown(self): # WebDriver'ı kapat self.driver.quit() def test_basarili_giris(self): # Form alanlarını doldur self.driver.find_element_by_id("username").send_keys("test_user") self.driver.find_element_by_id("password").send_keys("secure_password") # Giriş butonuna tıkla self.driver.find_element_by_id("login-button").click() # Sayfanın yüklenme süresi için bekle time.sleep(2) # Ana sayfaya yönlendirildik mi? self.assertEqual(self.driver.current_url, "http://example.com/dashboard") # Karşılama mesajı var mı? welcome_message = self.driver.find_element_by_id("welcome") self.assertIn("Hoş geldiniz", welcome_message.text)

Test Güdümlü Geliştirme (TDD)

Test Güdümlü Geliştirme (Test-Driven Development - TDD), önce testleri yazıp sonra bu testleri geçecek kodu yazma pratiğidir. TDD'nin temel döngüsü şu şekildedir:

  1. Kırmızı: Başarısız bir test yaz
  2. Yeşil: Testi geçecek kadar (ve sadece geçecek kadar) kod yaz
  3. Yeniden düzenle: Kodu temizle, tekrarları kaldır, daha okunabilir hale getir
# TDD örneği - Başlangıçta hata veren bir test yazalım import unittest class TestStringIslemleri(unittest.TestCase): def test_tersten_yaz(self): self.assertEqual(tersten_yaz("merhaba"), "abahrem") self.assertEqual(tersten_yaz("python"), "nohtyp") self.assertEqual(tersten_yaz(""), "") # Şimdi testi geçecek kodu yazalım def tersten_yaz(metin): return metin[::-1] # Bu aşamada test başarılı olacaktır if __name__ == '__main__': unittest.main()

Mock Nesneler

Python'da mock nesneler, unittest.mock modülü ile oluşturulabilir. Bu nesneler, dış bağımlılıkları (API çağrıları, veritabanı bağlantıları vb.) simüle etmek için kullanılır.

import unittest from unittest.mock import Mock, patch def api_veri_getir(url): # Gerçekte bir API'ye istek yapan fonksiyon import requests response = requests.get(url) return response.json() def kullanici_bilgileri(kullanici_id): # API'den kullanıcı bilgilerini getiren fonksiyon url = f"https://api.example.com/users/{kullanici_id}" veri = api_veri_getir(url) return f"{veri['ad']} {veri['soyad']}" class TestKullaniciBilgileri(unittest.TestCase): @patch('__main__.api_veri_getir') # api_veri_getir fonksiyonunu mockla def test_kullanici_bilgileri(self, mock_api): # Mock davranışını belirle mock_api.return_value = {"ad": "Ahmet", "soyad": "Yılmaz"} # Fonksiyonu test et sonuc = kullanici_bilgileri(123) # Doğrulamalar mock_api.assert_called_once_with("https://api.example.com/users/123") self.assertEqual(sonuc, "Ahmet Yılmaz")

Test Coverage (Kapsama)

Test coverage (kapsama), kodunuzun ne kadarının testler tarafından çalıştırıldığını ölçen bir metriktir. Python'da coverage.py kütüphanesi, kodunuzun test kapsamını analiz etmek için kullanılabilir.

# coverage.py kullanımı - Komut satırından # pip install coverage # coverage run -m pytest test_dosyasi.py # coverage report # coverage html # Detaylı HTML raporu # pytest ile coverage entegrasyonu # pip install pytest-cov # pytest --cov=paket_adi test_dosyasi.py

İyi Test Yazma Prensipleri

Etkili testler yazmak için aşağıdaki prensiplere dikkat edilmelidir:

  • FIRST prensipleri: Fast (Hızlı), Independent (Bağımsız), Repeatable (Tekrarlanabilir), Self-validating (Kendini doğrulayan), Timely (Zamanında)
  • Her test tek bir şeyi test etmelidir
  • Testler izole olmalı - Bir test diğer testlerin sonucunu etkilememeli
  • Testler deterministik olmalı - Her çalıştırmada aynı sonucu vermeli
  • Edge case'leri test edin - Sınır değerleri, geçersiz girişler, boş değerler vb.
  • Test adları açıklayıcı olmalı - Ne test ettiğini açıkça belirtmeli
# İyi isimlendirilmiş test örneği def test_kullanici_adi_bos_olamaz(): with pytest.raises(ValueError): kullanici_olustur(email="test@example.com", ad="") # Edge case test örneği def test_liste_toplami_bos_liste_icin_sifir_olmali(): self.assertEqual(liste_toplami([]), 0) # Bağımsız test örneği def test_veritabani_kayit_ekleme(): # Test başlangıcında temiz bir durum oluştur veritabani_temizle() # İşlemi gerçekleştir sonuc = kayit_ekle({"ad": "Test"}) # Doğrula assert sonuc == True assert kayit_say() == 1

  • Entegrasyon Testleri: Birimler arası entegrasyonu test etme yöntemlerini öğrenin.
